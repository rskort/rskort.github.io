CP2K Input Generator

Guided wizard for producing a readable CP2K input. Defaults follow common CP2K recommendations where the manual defines them. Inline links jump to the CP2K manual so you know what each keyword controls.

CP2K input tree GLOBAL DFT MD SUBSYS

Step 1 - Choose workflow and scope

Pick the type of run and system boundary conditions. See GLOBAL → RUN_TYPE and POISSON → PERIODIC.

Step 2 - Define system and upload structure

Upload an .xyz file (no coordinates leave placeholders). For bulk, provide cell edges (ABC). See SUBSYS → COORD and SUBSYS → CELL.

Only .xyz is supported. Parsed locally; nothing is uploaded.

Step 3 - Method and basis presets

Choose XC, basis/pseudopotential, cutoffs, and spin treatment. See DFT → XC, MGRID, and SCF.

Use Gamma for large cells; Monkhorst-Pack for small periodic cells.

Step 4 - Run parameters

Configure SCF or MD controls. See SCF and MOTION → MD.

EPS_SCF
MAX_SCF

Default EPS_SCF 1e-5 (CP2K default) and 50 SCF cycles.

Step 5 - Output settings

Control printed data and restart cadence. See GLOBAL → PRINT, DFT PRINT, and MOTION PRINT.

0 means do not print forces.

0 means do not print stress.

Review the summary, sanity checks, and preview. Use Copy/Download. All generation stays in-browser.
